이벤트

1. 이벤트 모델의 종류

1) 고전 이벤트 모델

요소 객체가 가지고 있는 속성에 이벤트 핸들러 연결
이벤트 객체를 제거할 수 있으나, 제거하고자 한다면 해당 이벤트 속성값에 null 할당

2) 인라인 이벤트 모델

요소 내부에 이벤트 속성으로 실행 내용을 작성
단순한 코드는 직접 입력해도 좋음
일반적으로는, scipt 태그에 정의된 함수를 호출하는 방식으로 작성

3) 표준 이벤트 모델

w3 에서 공식적으로 지정한 이벤트 모델
한번에 여러개의 이벤트 핸들러 설정이 가능

2. 이벤트가 발생한 요소 객체에 접근하는 방법

이벤트가 발생한 요소 객체는, 내부적으로 이벤트 발생 정보를 담고 있는 객체를 매개변수로 전달한다.
이벤트 객체를 통해서, 해당 요소 객체에 접근이 가능
콜백 함수의 종류에 따라 this로 접근이 가능 this나 매개변수로 받아와서 target등으로

3. 태그별 기본적으로 가지고 있는 이벤트 제거

기본 이벤트를 가지고 있는 요소
a : 클릭 시, href 속성에 작성된 링크로 이동하는 이벤트 보유 submit : 클릭 시, 입력 양식 제출 후 페이지를 다시 로딩하는 이벤트 보유

기본 이벤트가 있는 요소에 onclick 이벤트를 설정하면, 이 코드가 우선 작동하고, 기본 이벤트가 실행됨
태그에 기본적으로 설정되어 있는 이벤트를 제거하기 위해서는 이벤트 핸들러의 return 값을 false로 지정
